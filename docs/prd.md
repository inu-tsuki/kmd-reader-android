# KMD Reader Android 需求文档

> 项目阶段：课程项目第一阶段 PRD
> 文档状态：草案
> 最近更新：2026-05-07

## 1. 项目背景

KMD 是一种面向动态文本、舞台演出和交互式阅读体验的内容格式。当前 KMD 主要以 Web 编辑器形态存在，核心能力集中在语法解析、布局计算、特效管线和舞台渲染上。随着格式能力逐渐成型，KMD 需要一个更贴近普通读者的移动端入口，让创作内容不只停留在编辑器中，也可以作为作品被浏览、收藏和阅读。

KMD Reader Android 是 KMD 内容生态的移动端阅读宿主。它不在 Android 端重新实现 KMD 解析器，而是复用现有 Web Runtime 作为唯一解释核心，避免在 KMD 快速迭代阶段出现多套 parser、多套 layout 和多套渲染行为。Android 应用负责移动端产品体验，包括作品发现、预览流浏览、书架管理、阅读进度、本地导入、离线缓存、阅读设置和审核辅助工具。

把 KMD 搬上移动端，不应只是把 Web 阅读器缩小到手机屏幕。移动端更适合承载纵向滚动、短预览、预加载和轻量决策：用户可以像浏览短视频一样快速扫过作品摘要、评论摘要、属性信息和预览片段，但不会被强制开始完整观看。

当前并不存在真实的 KMD 创作者社区，因此本项目第一版以“模拟社区内容源 + 本地示例作品 + 本地审核队列”为基础实现完整阅读和审核流程，并为未来接入真实社区 API 预留数据结构和页面边界。

## 2. 产品定位

KMD Reader Android 是一款面向 KMD 创作内容的移动阅读与社区浏览应用。用户可以在应用中浏览 KMD 作品预览流，查看作品详情，打开作品进行阅读，并管理自己的收藏、阅读历史和本地导入内容。面向未来社区，本应用还提供评审志愿者入口，用于检查待上架脚本、查看运行诊断、进行轻度修改并给出审核结论。

产品关键词：

- KMD 内容阅读
- 移动端作品发现流
- 短预览与预加载
- 评论摘要和属性展示
- 书架与阅读进度
- 本地文件导入
- 评审志愿者审核工具
- 面向未来社区的接口预留
- 单核心、多宿主架构

## 3. 产品目标

### 3.1 课程项目目标

- 完成一个具备完整产品闭环的 Android 阅读应用原型。
- 使用 Android 原生能力实现预览流、详情、书架、设置、持久化、文件导入和审核工作台等功能。
- 通过 WebView 承载 KMD Web Runtime，完成 KMD 文件阅读能力。
- 通过脚本检查、属性展示和审核队列，体现 Android 端不只是阅读套壳，而是社区内容流转工具。
- 提供清晰的 PRD、架构设计、数据模型和测试说明，支撑课程结项展示。

### 3.2 KMD 生态目标

- 验证 KMD 内容在移动端消费场景中的可行性。
- 为未来 KMD 社区、作品发布和移动端阅读体验建立产品骨架。
- 定义 KMD 社区的基础内容生命周期：提交、检查、审核、上架、浏览和阅读。
- 在不制造第二套核心实现的前提下，为 KMD Reader 提供跨平台宿主方案。

## 4. 目标用户

### 4.1 普通读者

对动态文本、视觉小说、互动诗歌、实验性阅读内容感兴趣，希望通过手机浏览和阅读 KMD 作品。

核心诉求：

- 通过滑动预览流快速找到感兴趣的作品。
- 在不进入完整阅读前了解作品风格、长度、标签和社区反馈。
- 阅读体验稳定、流畅、不会丢失进度。
- 可以收藏作品并继续阅读。

### 4.2 KMD 创作者

使用 KMD 编辑器创作内容，希望在移动端预览作品的实际阅读效果，或把作品展示给他人。

核心诉求：

- 导入本地 KMD 文件进行阅读预览。
- 阅读器表现与 Web 编辑器尽量一致。
- 能看到标题、作者、标签、简介等作品信息。
- 在未来社区中了解脚本检查结果、修改建议和上架状态。

### 4.3 评审志愿者

未来 KMD 社区中的轻量审核参与者，负责判断提交脚本是否具备上架条件。评审志愿者不承担完整编辑工作，而是使用检查工具和预览工具发现问题，必要时进行小范围元信息或文本修正。

核心诉求：

- 快速查看待审核脚本的属性、摘要和预览效果。
- 获取 parser、layout、effect 和资源检查结果。
- 对明显的小问题进行轻度修改，例如标题、标签、简介、内容提示或少量文本补丁。
- 给出审核结论：通过、退回修改、拒绝上架。

### 4.4 课程评审者

关注项目是否具备明确需求、合理技术方案、实际工作量和可演示成果。

核心诉求：

- 产品定位清晰。
- 功能范围可落地。
- Android 端工作量充分，不只是简单网页套壳。
- 技术风险有分析和应对策略。

## 5. 使用场景

### 5.1 通过预览流浏览社区作品

用户打开应用后进入发现流，看到由模拟社区数据提供的纵向作品预览卡片。当前卡片展示标题、作者、标签、预计阅读时长、评论摘要和脚本属性，应用预加载当前作品和下一部作品的预览资源。用户可以滑动切换作品，停留时播放短预览，点击后才进入完整作品详情或阅读器。

### 5.2 查看作品属性和社区摘要

用户在预览卡片或作品详情页查看作品属性，包括类型、标签、时长、动态效果强度、是否包含外部资源、适读提示和评论摘要。评论摘要用于帮助用户判断是否继续阅读，但不要求用户打开完整评论区。

### 5.3 阅读 KMD 作品

用户在作品详情页点击“开始阅读”，应用打开阅读器页面。Android 侧将 KMD 文本、主题、字号和上次阅读进度传入 Web Runtime，Web Runtime 负责解析与渲染。阅读过程中，阅读器将进度回传给 Android 侧保存。

### 5.4 管理书架

用户可以收藏作品，收藏后的作品出现在书架页。书架页同时展示最近阅读记录，方便用户继续未完成作品。

### 5.5 本地导入 KMD 文件

创作者或测试用户可以从手机存储中选择 `.kmd` 文件导入应用。导入后，应用将文件作为本地作品保存，并允许用户从详情页或书架页打开阅读。

### 5.6 审核待上架脚本

评审志愿者进入审核工作台，查看模拟待审核队列。点击某个脚本后，应用展示脚本属性、预览播放、检查结果和问题列表。志愿者可以进行轻度修改，保存修改记录，并给出通过、退回修改或拒绝上架的审核结论。

### 5.7 调整阅读偏好

用户可以在设置页调整阅读主题、字号、是否自动保存进度等偏好。设置项通过本地持久化保存，并在下次进入阅读器时自动生效。

## 6. 社区形态与内容生命周期

KMD Reader Android 在课程项目中使用本地 mock 数据模拟社区，但需求上需要提前定义未来社区的基本样子。社区不是只有作品展示列表，而是一条围绕 KMD 脚本质量和移动端消费体验建立的内容流转链路。

### 6.1 社区角色

| 角色       | 说明                                                      |
| ---------- | --------------------------------------------------------- |
| 读者       | 浏览预览流、查看属性和摘要、收藏并阅读作品                |
| 创作者     | 创作 KMD 脚本，提交作品并查看检查结果                     |
| 评审志愿者 | 检查待上架脚本，进行轻度修改并给出审核结论                |
| 平台维护者 | 维护规则、分类、推荐策略和审核配置，课程 MVP 不实现该后台 |

### 6.2 内容状态

```text
本地草稿
  -> 提交审核
  -> 自动检查
  -> 志愿者审核
  -> 通过上架
  -> 进入发现流
```

异常状态：

- `needs_changes`：需要创作者修改后重新提交。
- `rejected`：不适合上架或存在严重问题。
- `archived`：历史版本归档。

### 6.3 评审原则

评审志愿者的工作重点是判断作品是否“可以被读者稳定浏览和阅读”，而不是替创作者重写作品。

评审需要关注：

- 脚本能否被当前 KMD Runtime 解析。
- 是否存在明显布局、特效或资源错误。
- 预览片段是否可以正常播放。
- 标题、简介、标签和内容提示是否完整。
- 作品是否适合进入公开发现流。

### 6.4 轻度修改边界

允许的轻度修改：

- 修改标题、简介、标签、分类和内容提示。
- 修正少量明显错别字或格式瑕疵。
- 添加审核备注和退回修改建议。
- 调整预览片段的起止位置或展示文案。

不允许的修改：

- 重写作品结构。
- 大范围改写正文。
- 修改 KMD 核心语法含义。
- 在 Android 端实现完整 KMD 编辑器。

## 7. 功能范围

### 7.1 MVP 必须实现

| 模块       | 功能             | 说明                                                 |
| ---------- | ---------------- | ---------------------------------------------------- |
| 发现流     | 纵向预览流       | 使用本地 mock 数据模拟社区作品，以移动端滑动卡片呈现 |
| 发现流     | 预加载策略       | 预加载当前卡片和下一张卡片的预览内容，避免滑动后空白 |
| 发现流     | 评论摘要         | 展示本地 mock 的评论摘要、亮点和争议点               |
| 发现流     | 作品属性展示     | 展示时长、标签、动态效果强度、资源依赖和适读提示     |
| 发现流     | 搜索             | 按标题、作者、标签搜索                               |
| 发现流     | 分类/标签筛选    | 支持基础分类筛选                                     |
| 作品详情   | 基本信息展示     | 标题、作者、简介、封面、标签、字数或预计阅读时长     |
| 作品详情   | 开始/继续阅读    | 根据阅读历史决定按钮状态                             |
| 书架       | 收藏作品         | 展示用户收藏的作品                                   |
| 书架       | 最近阅读         | 展示阅读过的作品与进度                               |
| 阅读器     | KMD 渲染         | 通过 WebView 加载 KMD Web Runtime                    |
| 阅读器     | 阅读进度保存     | Web Runtime 回传进度，Android 侧持久化               |
| 阅读器     | 阅读设置应用     | 支持字号、主题等基础参数                             |
| 本地导入   | 导入 `.kmd` 文件 | 将文件加入本地作品库                                 |
| 审核工作台 | 待审核队列       | 使用本地 mock 数据展示待审核脚本                     |
| 审核工作台 | 脚本检查         | 展示 parser、layout、effect、资源和性能提示          |
| 审核工作台 | 审核预览         | 在受控 WebView 中播放短预览，辅助判断是否可上架      |
| 审核工作台 | 轻度修改         | 支持元信息修改和小范围文本补丁，不实现完整编辑器     |
| 审核工作台 | 审核结论         | 支持通过、退回修改、拒绝上架，并保存审核备注         |
| 设置       | 阅读偏好         | 保存主题、字号等偏好                                 |
| 数据层     | 本地持久化       | 保存收藏、历史、导入作品、审核记录和设置             |

### 7.2 可选增强

- 作品详情中的作者主页。
- 作品评分或点赞展示。
- 阅读器夜间模式。
- 最近导入文件列表。
- 错误日志页面，用于展示 KMD 解析或渲染失败信息。
- 审核差异视图，用于比较原脚本和轻度修改后的版本。
- 真实评论数据汇总成评论摘要。
- 自动生成 KMD 预览片段或封面动图。
- 网络 API mock 层，为未来真实社区接口做替换准备。

### 7.3 暂不实现

- 用户注册与登录。
- 真实作品上传与发布。
- 完整评论区、关注、私信等社区互动。
- 推荐算法。
- 付费、打赏或会员体系。
- Android 端完整 KMD 编辑器。
- 大范围脚本改写和多人协同审核。
- Kotlin 原生 KMD parser 或原生 KMD renderer。

## 9. 核心用户流程

### 9.1 预览流浏览流程

```text
打开应用
  -> 进入发现流
  -> 当前卡片播放短预览
  -> 查看作品属性和评论摘要
  -> 滑动到下一张预加载卡片
  -> 进入作品详情
  -> 点击开始阅读
  -> 阅读器加载 KMD 内容
  -> 阅读过程中保存进度
  -> 返回书架后可继续阅读
```

### 9.2 本地作品导入流程

```text
进入本地导入
  -> 选择 .kmd 文件
  -> 应用读取文件内容
  -> 生成本地作品记录
  -> 进入作品详情
  -> 打开阅读器
```

### 9.3 审核志愿者流程

```text
进入审核工作台
  -> 打开待审核脚本
  -> 查看自动检查结果
  -> 播放短预览
  -> 查看属性、摘要和问题列表
  -> 进行轻度修改或填写审核备注
  -> 给出通过/退回/拒绝结论
  -> 保存审核记录
```

### 9.4 收藏流程

```text
进入作品详情
  -> 点击收藏
  -> 写入本地收藏记录
  -> 作品出现在书架
  -> 再次点击可取消收藏
```

| fontScale        | Float   | 字号缩放         |
| autoSaveProgress | Boolean | 是否自动保存进度 |

## 11. WebView 桥接需求

KMD Reader Android 采用单核心、多宿主架构。Android 端不拥有 KMD parser、layout 或 renderer 的第二份实现，而是通过 WebView 加载打包后的 KMD Reader Web Runtime。

### 11.1 Android 到 Web Runtime

Android 侧需要向 Web Runtime 传递：

- KMD 文本内容或内容 URL。
- 作品 ID。
- 运行模式：`preview` / `reader` / `inspection`。
- 阅读主题。
- 字号参数。
- 上次阅读进度。
- 预览起止位置。
- 审核检查开关。
- 可选调试开关。

### 11.2 Web Runtime 到 Android

Web Runtime 需要向 Android 侧回传：

- 当前阅读进度。
- 当前章节或位置标识。
- 作品标题等解析出的元信息。
- 作品属性统计，例如指令数量、特效强度和外部资源数量。
- 预览加载完成事件。
- 脚本检查问题列表。
- 解析失败或渲染失败错误。
- 阅读器加载完成事件。

### 11.3 错误处理

当 Web Runtime 解析或渲染失败时，Android 侧应展示错误状态，并提供返回、重试和查看错误摘要的入口。错误信息需要保存到日志中，便于课程演示和后续调试。

## 12. 非功能需求

### 12.1 性能

- 作品列表首屏加载应在 1 秒内完成。
- 阅读器页面应展示加载状态，避免空白等待。
- 预览流只应预加载当前卡片和相邻卡片，避免同时启动多个 KMD Runtime。
- 预览播放应优先使用短片段或轻量模式，减少移动端耗电和卡顿。
- 本地 mock 作品和导入作品应支持离线访问。

### 12.2 一致性

- KMD 文件解释结果应以 Web Runtime 为准。
- Android 端不得维护独立 KMD parser，避免行为分叉。
- 若 Web 编辑器和 Android 阅读器表现不一致，应优先在 KMD Web Runtime 中定位问题。

### 12.3 可维护性

- Android 应用层、数据层、阅读器桥接层应分离。
- Mock 数据源应可替换为真实远程 API。
- Web Runtime 构建产物应有明确版本标识。
- 审核规则和脚本检查结果应独立建模，避免写死在页面展示逻辑中。

### 12.4 可用性

- 无作品、无收藏、导入失败、解析失败等状态应有明确提示。
- 阅读进度应自动保存，避免用户丢失阅读位置。
- 主要阅读流程不依赖网络。
- 预览流不应强制用户进入完整作品阅读。
- 审核工具应明确区分“检查结果”“轻度修改”和“审核结论”。

### 12.5 审核可追溯性

- 每次审核结论应保存时间、评审身份和备注。
- 轻度修改应保留修改前后的摘要，便于回溯。
- 审核工具应避免隐藏严重错误，`error` 级问题默认不允许直接通过。

推荐技术栈：

- Kotlin
- Jetpack Compose
- Navigation Compose
- ViewModel
- Room
- DataStore

推荐目录形态：

## 15. 风险与应对

| 风险                       | 影响                           | 应对                                                                     |
| -------------------------- | ------------------------------ | ------------------------------------------------------------------------ |
| KMD 格式仍在快速变化       | Android 阅读器可能频繁适配     | 保持单一 Web Runtime 核心，Android 只消费构建产物                        |
| WebView 被误认为工作量不足 | 影响课程评价                   | Android 侧实现完整产品骨架、预览流、数据层、持久化、导入流程和审核工作台 |
| Reader Runtime 尚未独立    | 接入初期需要临时打包方案       | 先复制构建产物，后续抽出 `reader-runtime-web` / `@kmd/reader-runtime-web` |
| 移动端性能不稳定           | 阅读体验受影响                 | 限制 MVP 样例复杂度，增加加载状态和错误提示                              |
| 预览流同时加载过多内容     | 导致卡顿、耗电或 WebView 崩溃  | 只预加载当前和相邻卡片，限制预览片段长度                                 |
| 审核工具范围膨胀           | 项目变成移动端编辑器，难以收尾 | 限定轻度修改边界，复杂修改退回创作者                                     |
| 没有真实社区               | 作品发现缺少真实数据           | 使用本地 mock 数据模拟社区，保留 API 替换边界                            |

## 16. 验收标准

课程第一阶段 PRD 验收：

- 文档说明项目背景、目标用户、核心场景和功能范围。
- 文档明确 MVP 与暂不实现内容。
- 文档解释为什么不进行纯 Android 端核心移植。
- 文档定义未来社区、预览流和评审志愿者角色。
- 文档给出页面结构、数据模型和技术方案。
- 文档能支撑后续原型设计与 Android 项目开发。

后续实现阶段验收：

- 用户可以通过纵向预览流浏览作品并进入详情页。
- 用户可以在不开始完整观看的情况下查看短预览、评论摘要和作品属性。
- 用户可以打开 KMD 作品阅读。
- 用户可以收藏作品并在书架中找到。
- 用户可以继续上次阅读进度。
- 用户可以导入本地 `.kmd` 文件。
- 评审志愿者可以查看待审核队列、脚本检查结果和审核预览。
- 评审志愿者可以进行轻度修改并保存审核结论。
- 阅读器出错时应用能展示错误状态。

## 17. 术语表

| 术语           | 说明                                                          |
| -------------- | ------------------------------------------------------------- |
| KMD            | 项目自定义的动态文本内容格式                                  |
| KMD Core       | KMD 的解析、布局、效果和渲染核心能力                          |
| Web Runtime    | 在 Web 环境中运行的 KMD 阅读器内核                            |
| 宿主           | 承载 KMD Runtime 的应用环境，例如 Web 编辑器或 Android Reader |
| Mock 社区      | 使用本地静态数据模拟的作品社区                                |
| 单核心、多宿主 | 只维护一套 KMD 解释核心，由多个应用宿主复用                   |
| 预览流         | 面向移动端滑动浏览的作品短预览信息流                          |
| 评审志愿者     | 负责检查待上架 KMD 脚本并给出审核结论的社区角色               |
| 脚本检查       | 对 KMD 脚本进行解析、布局、特效、资源和性能风险检查           |
| 轻度修改       | 不改变作品结构的小范围元信息或文本修正                        |
